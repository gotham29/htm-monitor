# HTM-Monitor Demo Platform

A self-contained web demo for showcasing HTM-Monitor to prospective clients.
No Node.js or build step required — pure Python backend, vanilla JS + Plotly.js frontend.

---

## Quick Start

### 1. Install dependencies

From the repo root:

```bash
pip install -r demo/requirements_demo.txt
```

### 2. Build the demo data artifact

This merges `outputs/powergrid_ca/run.csv` with the raw signal CSVs into a
single columnar JSON file that the server can stream to the browser.

```bash
# From the repo root:
python demo/build_demo_data.py
```

Expected output:
```
[build] Loading manifest: outputs/powergrid_ca/run.manifest.json
[build] Loading signal CSVs from: data/powergrid_ca/caiso_2019fit_2020_2022_pad0p00
  Loaded demand: 26,255 rows
  Loaded net_generation: 26,255 rows
  ...
[build] Loading run.csv: outputs/powergrid_ca/run.csv
  Loaded 26,255 timesteps
[build] Writing output: demo/static/demo_data/powergrid_ca.json
[build] Done. Output size: ~8.0 MB  (26,255 steps)
```

If your paths differ, pass them explicitly:

```bash
python demo/build_demo_data.py \
  --run-csv   outputs/powergrid_ca/run.csv \
  --manifest  outputs/powergrid_ca/run.manifest.json \
  --data-dir  data/powergrid_ca/caiso_2019fit_2020_2022_pad0p00 \
  --out       demo/static/demo_data/powergrid_ca.json
```

### 3. Start the server

```bash
python demo/server.py
```

Then open **http://localhost:8000** in your browser.

Default login credentials:
- **Username:** `demo`
- **Password:** `htm-monitor-2024`

## Configuration

All config is via environment variables:

| Variable         | Default           | Description                        |
|------------------|-------------------|------------------------------------|
| `HTM_DEMO_USER`  | `demo`            | Login username                     |
| `HTM_DEMO_PASS`  | `htm-monitor-2024`| Login password                     |
| `HTM_SESSION_TTL`| `86400`           | Session duration in seconds (24h)  |
| `HTM_PORT`       | `8000`            | Server port                        |
| `HTM_HOST`       | `0.0.0.0`         | Server bind host                   |

Example — change credentials before a client demo:

```bash
HTM_DEMO_USER=client HTM_DEMO_PASS=SecurePass2024 python demo/server.py
```

## Adding a New Use Case

1. Add an entry to `USECASE_META` in `build_demo_data.py` with your signals, groups, events, and source CSV mapping.
2. Run `build_demo_data.py --usecase your_usecase_id --out demo/static/demo_data/your_usecase_id.json`.
3. The server auto-discovers all `*.json` files in `demo/static/demo_data/` — no server changes needed.

## Demo Walkthrough

The app has three screens:

**Login** → Enter credentials → **Catalog** → Select a use case → **Replay Viewer**

In the Replay Viewer:
- **Left sidebar**: System alert status (pulsing badge when active), group tiles with warmth bars (Demand / Generation / Imbalance), named event list with "Jump to event" buttons, current timestamp display
- **Top chart**: Raw signal values (5 traces, colored by group), with event regions shaded
- **Bottom chart**: Per-model anomaly probability (0–1), threshold line, alert shading
- **Playback controls**: Play/Pause, speed selector (0.5×–8×), window selector (7/14/30/90 days or full data)
- **Jump buttons**: Click any named event to snap the playhead to that moment

The playback efficiently moves only the x-axis window over pre-rendered static traces — no data re-fetching during replay.
